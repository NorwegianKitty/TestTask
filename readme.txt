

Для аргументов -p и -o, в случае невозможности установки значений их параметров в качестве префикса или пути соотвественно, будут использованы значения по умолчанию: Для имен: integers.txt, floats.txt, strings.txt и текущая директория для пути.
==================

1. Аргумент '-p'
● Тк параметры после аргументов -p могут содержать значения, вида -a, для префикса, или .txt не вижу возможности различить пропущенный параметр и следущий аргумент, а значит программа выполниться так, словно, если следущий аргумент есть, он, если пройдет череду проверок, будет являться параметром для предшествующего аргумента и ничем иным уже быть не сможет.
Те аргументы вида '-p -a' будут восприняты как аргумент '-p' с параметром для него '-a', что задаст префикс именам файлов как '-a', при этом режим добавления включен не будет.
● Так же, проверка допустимых имен файлов создана для Windows.
==================

2. Аргумент '-o'

==================


2. Аргументы '-s' и '-f'
● Касаемо статистики, ввиду малой масштабности задачи решено было, для простоты использовать стандарнтые средства сбора данных, не вынося это в отдельную базу данных.
